# 🎯 СТАНДАРТ: Единый язык проекта Taskorium

## 📌 Метаданные

- **Версия**: 1.0
- **Дата введения**: 2027-11-30
- **Статус**: Обязателен к соблюдению

## 🎯 Цель

Установить единую терминологию для всех участников проекта для исключения разночтений и улучшения коммуникации.

## 📖 Область действия

Все документы, код, коммиты, обсуждения проекта **Taskorium**.

## 🏗️ **СЛОВАРЬ ТЕРМИНОВ**

### **🎯 ОСНОВНЫЕ СУЩНОСТИ**

#### **User (Пользователь)**

**Термин**: User / Пользователь
**Контекст**: Система в целом
**Определение**: Физическое лицо, зарегистрированное в системе Taskorium
**Синонимы**: ❌ ПользовательСистемы, ❌ Аккаунт
**Анти-паттерны**: ❌ "Юзер", ❌ "Учетка"
**Примеры**:
✅ "User создает Task в Project"
✅ "User назначается исполнителем Task"
❌ "Юзер создает таску в проекте"

**Термин**: Task / Задача
**Контекст**: Управление задачами  
**Определение**: Единица работы, которую необходимо выполнить в рамках Project
**Атрибуты**: title, description, status, priority, assignee
**Примеры**:
✅ "Task перемещается по статусам Workflow"
✅ "Task может иметь Attachment"
❌ "Тикет", ❌ "Ишью", ❌ "Таска"

**Термин**: Sprint / Спринт
**Контекст**: Agile планирование
**Определение**: Фиксированный временной интервал для выполнения набора Task
**Атрибуты**: startDate, endDate, goal, velocity
**Примеры**:
✅ "Sprint содержит selected Task"
✅ "Sprint имеет duration 2 недели"
❌ "Итерация", ❌ "Период"

#### Project Roles (роли в проекте)

**Project Owner** → Владелец проекта (создатель, полные права)
**Project Admin** → Администратор проекта (управление участниками)
**Project Member** → Участник проекта (создание/редактирование задач)  
**Project Viewer** → Наблюдатель (только чтение)

#### System Roles (роли в системе)

**System Administrator** → Администратор системы (глобальные права)
**Regular User** → Обычный пользователь (стандартные права)

**Пример**:
✅ "System Administrator может управлять всеми Project"

#### Task Statuses (статусы задачи)

**OPEN** → Задача создана, но не начата
**IN_PROGRESS** → Задача в работе
**CODE_REVIEW** → Задача на код-ревью
**TESTING** → Задача на тестировании
**DONE** → Задача завершена
**CANCELLED** → Задача отменена

**Использование**:
✅ "Task переходит в статус IN_PROGRESS"
❌ "Задача в состоянии 'в работе'"

#### Attachment System (файлы и вложения)

**Attachment** → Файл, прикрепленный к Task или Project
**File Metadata** → Информация о файле (название, размер, тип)
**Thumbnail** → Уменьшенная копия изображения для preview

#### Notification (Уведомление)

**Термин**: Notification / Уведомление
**Контекст**: Система оповещений
**Определение**: Сообщение от системы пользователю о произошедшем событии
**Атрибуты**: type, recipient, subject, body, status, created_at
**Примеры**:
✅ "System отправляет Notification о изменении статуса Task"
✅ "User получает Notification в реальном времени"
❌ "Система шлет нотификацию", ❌ "Юзер получает алерт"

**Термин**: Notification Channel / Канал уведомлений
**Контекст**: Доставка уведомлений  
**Определение**: Способ доставки Notification пользователю
**Типы**: EMAIL, PUSH, TG
**Примеры**:
✅ "User настраивает preferred Notification Channel"
✅ "System отправляет Notification через multiple channels"
❌ "Способ отправки уведомлений", ❌ "Тип нотификации"

#### Type of task notifications (типы уведомлений)

**Task Assigned Notification** → Уведомление о назначении на задачу
**Task Status Changed Notification** → Уведомление об изменении статуса
**Task Mention Notification** → Уведомление об упоминании в задаче
**Task Due Soon Notification** → Уведомление о приближении дедлайна
**Task Overdue Notification** → Уведомление о просрочке задачи

**Примеры**:
✅ "User получает Task Assigned Notification при назначении на Task"
✅ "System отправляет Task Due Soon Notification за 24 часа до due_date"

#### Notification Preferences (Настройки уведомлений)

**Термин**: Notification Preferences / Настройки уведомлений
**Контекст**: Управление подписками на уведомления
**Определение**: Персональные настройки пользователя о том, какие уведомления и каким способом получать
**Атрибуты**: user_id, notification_type, channel, enabled
**Примеры**:
✅ "User настраивает Notification Preferences в личном кабинете"
✅ "System учитывает Notification Preferences при отправке уведомлений"
❌ "Настройки нотификаций", ❌ "Преференции уведомлений"

#### Notification Dispatch (Отправка уведомлений)

**Термин**: Notification Dispatch / Отправка уведомлений
**Контекст**: Процесс доставки уведомлений
**Определение**: Процесс выбора канала и отправки уведомления получателю
**Примеры**:
✅ "System выполняет Notification Dispatch через выбранные channels"
✅ "Notification Service обрабатывает Notification Dispatch асинхронно"
